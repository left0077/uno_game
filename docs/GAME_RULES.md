# 🃏 Uno 游戏规则文档

> 版本：v1.0  
> 更新日期：2026-03-15

---

## 一、游戏概述

Uno 是一款经典的卡牌游戏，支持 **2-8人** 同时在线对战，真人玩家与 AI 机器人可混合参与。

---

## 二、卡牌组成

一副完整的 Uno 牌共 **108张**：

### 2.1 数字牌（76张）

| 颜色 | 0 | 1-9（各2张） | 小计 |
|------|---|-------------|------|
| 🔴 红色 | 1张 | 各2张 | 19张 |
| 🟡 黄色 | 1张 | 各2张 | 19张 |
| 🟢 绿色 | 1张 | 各2张 | 19张 |
| 🔵 蓝色 | 1张 | 各2张 | 19张 |

### 2.2 功能牌（24张）

| 类型 | 图标 | 效果 | 数量 |
|------|------|------|------|
| 跳过 | 🚫 | 下家跳过回合 | 各2张×4色 = 8张 |
| 反转 | ↩️ | 改变出牌顺序 | 各2张×4色 = 8张 |
| +2 | +2 | 下家摸2张并跳过 | 各2张×4色 = 8张 |

### 2.3 万能牌（8张）

| 类型 | 图标 | 效果 | 数量 |
|------|------|------|------|
| 变色 | 🌈 | 出牌者指定下一种颜色 | 4张 |
| +4 | +4 | 指定颜色，下家摸4张并跳过 | 4张 |

## 三、游戏流程

```
┌─────────────────────────────────────────────────────────────┐
│  ① 创建房间  →  ② 等待玩家  →  ③ 开始游戏  →  ④ 轮流出牌     │
│     ↓              ↓              ↓              ↓          │
│  房主创建       1-8人/AI      每人发7张牌    按顺序出牌      │
│  4位房间号**    可随时加AI    翻开首张牌*    匹配颜色/数字

** **房间号生成规则**：4位随机数字（1000-9999），若冲突则重新生成  
* **首张牌翻开规则**：
  - 翻开牌堆顶第一张牌作为起始牌
  - 若为万能牌（变色/+4）：放回牌堆，重新洗牌，最多重洗 **3次**
  - 若3次后仍翻到万能牌：**强制指定为红色数字0**（虚拟牌，不占用牌堆）   │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    ⑤ 游戏结束 → 展示排名 → 再来一局/解散
```

## 四、出牌规则

### 4.1 基本规则

1. **轮流出牌**：按顺时针方向（可被反转牌改变）
2. **匹配规则**：出的牌必须与弃牌堆顶部牌**颜色相同**或**数字/类型相同**
3. **万能牌**：可随时出牌，不受颜色限制
4. **无牌可出**：必须从牌堆摸1张牌
   - 若摸到的牌可出，可选择立即打出或保留（ optional ）
   - 若选择保留，回合结束

5. **牌堆可见性**：
   - **牌堆顶**：不可见，只显示剩余张数
   - **弃牌堆顶**：对所有玩家可见（显示最上面一张牌）
6. **牌堆耗尽**：当牌堆摸完时，将弃牌堆（保留最上面一张）洗匀作为新牌堆
   - **极端情况**：若牌堆和弃牌堆均耗尽（理论极端情况），游戏强制结束，剩余手牌最少者获胜
7. **回合限时**：
   - **开始时机**：轮到自己时（上家操作完成），**立即**开始2分钟倒计时
   - **前端显示**：显示倒计时进度条，最后30秒变红，最后10秒音效提醒
   - **超时处理**：自动从牌堆摸1张牌，回合结束
8. **摸牌后出牌限制**：
   - 若摸到的牌**可出**：可选择立即打出或保留（ optional ）
   - 若摸到的牌**不可出**：必须保留，回合结束
   - **超时摸牌**：无论摸到的是什么，均**不可立即打出**，回合直接结束
9. **手牌上限**：
   - 理论无上限，UI采用**横向滚动**或**堆叠显示**适配
   - 超过20张时显示提示"手牌过多，建议尽快出牌"

### 4.2 喊 Uno

#### 正确时机
- **标准流程**：玩家打出倒数第2张牌后，手中**剩余1张牌**时，必须立即点击 **"UNO!"** 按钮
- **顺序要求**：必须是**先出牌，后喊UNO**（出倒数第2张 → 手牌剩1张 → 喊UNO）

#### 质疑规则
- **质疑时间窗口**：从该玩家打出倒数第2张牌（手牌变为1张）开始，到**下家完成出牌/摸牌操作**为止
- **质疑按钮可见性**：所有玩家可见"质疑"按钮，但仅在**有效质疑窗口**内可点击
- **质疑对象**：其他玩家可在此期间点击"质疑"按钮
- **多人质疑**：只接受**第一个**到达服务器的质疑，其他质疑提示"已被质疑"
- **质疑成功**：被质疑者未在窗口期内喊UNO，罚摸 **2张牌**
- **质疑失败**：被质疑者已正确喊UNO，质疑者罚摸 **1张牌**（可选惩罚）
- **窗口关闭**：下家完成操作后，不再接受质疑

### 4.3 功能牌效果

| 卡牌 | 效果说明 | 使用限制 |
|------|---------|---------|
| **跳过** | 下家失去本轮出牌机会 | 需颜色匹配 |
| **反转** | 出牌顺序反转（顺时针↔逆时针），**2人局时等同于跳过** | 需颜色匹配 |
| **+2** | 下家必须摸2张牌并跳过回合 | 需颜色匹配 |
| **变色** | 出牌者选择下一种颜色（红/黄/绿/蓝） | **无限制**，随时可出 |
| **+4** | 出牌者选择颜色，下家摸4张并跳过 | **仅当手牌中没有与当前弃牌颜色匹配的牌时可用** |

#### 多牌同出规则（可选）
- **条件**：手牌中有多张**相同数字**的牌（颜色可以不同）
- **操作**：可一次性打出所有相同数字的牌
- **第一张匹配**：多牌同出的第一张必须与弃牌堆顶**颜色或数字匹配**
- **顺序**：按出牌顺序依次生效（如出两个+2，下家摸4张并跳过）
- **多牌同出排序规则**：玩家可选择出牌顺序，或按系统默认（建议按颜色或类型分组）
- **多牌同出功能牌混合**：若同时出+2和跳过，按出牌顺序依次执行；+2使下家摸牌跳过，跳过则影响下下家
- **多牌同出中的反转**：若包含反转牌，**反转效果在所有牌打出后生效**（不影响当前多牌同出过程），影响下**一轮**的出牌顺序
- **最后一张**：若多牌同出后手牌为空，只需在最后一张牌打出后喊UNO
- **多牌同出时喊UNO时机**：
  - 若多牌同出包含倒数第2张和最后1张牌，在打出倒数第2张后（手牌剩2张时）**不需要**喊UNO
  - 在打出最后一张牌后（手牌剩1张，即将出完），**立即**喊UNO
  - 质疑窗口：从最后一张牌打出后开始，到游戏结束（或下家操作，但游戏会立即结束）
- **抢打+多牌同出**：抢打成功后，若手中有多个相同数字牌，可继续多牌同出
- **多牌同出+叠加**：多牌同出多张+2/+4时，可触发叠加，下家需摸取累计张数

#### +4 使用条件（宽松规则）
- **推荐策略**：当手牌中有与当前弃牌颜色匹配的牌时，建议优先出颜色匹配的牌
- **允许使用**：即使手牌中有匹配颜色的牌，也可以使用+4（不再强制验证）
- **风险提示**：频繁在没有必要的情况下使用+4可能引起其他玩家不满 😄
- ~~**系统验证**：后端不再强制验证玩家手牌是否满足+4使用条件~~
- ~~**违规处理**：~~ 不再强制收回+4或罚摸

### 4.5 抢打出牌规则（可选）

- **触发条件**：其他玩家出牌后，若你手中有**完全相同**的牌（数字和颜色都相同），可立即抢打出牌
- **效果**：跳过中间等待的玩家，轮到你继续出牌
- **限制**：
  - 仅数字牌可抢打（功能牌和万能牌不可抢打）
  - 必须在**2秒**内点击抢打按钮，超时失效
  - 多人同时抢打时，以服务器时间戳为准，先到达者成功
- **抢打后多牌同出**：抢打成功后，若手中有其他与抢打牌**相同数字**的牌，可一并打出（多牌同出）
- **抢打喊UNO**：抢打后手牌剩1张时，需在2秒抢打窗口内或出牌后立即喊UNO
- **抢打+多牌同出+UNO**：
  - 抢打成功后多牌同出，包含倒数第2张和最后1张时
  - 在打出最后一张后喊UNO
  - **质疑窗口**：从最后一张牌打出后开始，到游戏结束
  - 由于游戏会立即结束，质疑窗口极短（ practically 只接受立即质疑）
- **抢打与叠加**：
  - 抢打一张+2/+4可触发叠加
  - 抢打者可以选择"继续叠加"：从手牌再出一张+2/+4（**必须与抢打的牌同类型**，+2接+2，+4接+4）
  - 或选择"停止叠加"：轮到原下家（按抢打后的新顺序）选择接牌或摸牌
  - 叠加计数器累加，直到某玩家无法接牌时摸取累计张数
  - **抢打触发的叠加，不视为抢打者的新回合**，只是叠加链的延续
- **抢打后的回合**：抢打成功后，**抢打者获得下一轮回合权**（正常出牌顺序被中断）
- **连续抢打**：A抢打成功后，其他玩家**不能**抢打A出的牌（避免无限循环）
- **多牌同出过程中的抢打**：多牌同出视为**一次性操作**，出牌过程中**不可被抢打**。抢打只针对**单张出牌**触发
- **抢打计时器**：抢打不影响正常回合计时，抢打成功后重新开始该玩家的2分钟倒计时
- **抢打与反转**：若上家出的是反转牌，抢打**不会**改变反转效果，抢打者出牌后反转效果继续生效
- **抢打与质疑窗口**：
  - 玩家A出倒数第2张后，质疑窗口开启（针对A的UNO）
  - 玩家B抢打成功，**A的质疑窗口保持开启**，其他玩家仍可在窗口期内质疑A
  - 抢打者B的质疑窗口从B打出倒数第2张时开启
  - 如果A在抢打后立即出完牌（被抢打前只剩1张），质疑窗口立即关闭
- **叠加时的抢打**：+2叠加时，有人出+2后，其他玩家**不能**抢打（因为叠加期间只有下家可以接牌）

### 4.4 叠加规则（简化版）

- **+2 只能接 +2**（同颜色或不同颜色）
- **+4 只能接 +4**
- **不可混叠**：+2 不能接 +4，反之亦然
- 连续叠加时，最后无法接牌的玩家摸取累计张数
- **叠加上限**：无上限，理论上可无限叠加（实际受牌堆限制）
- **叠加计时**：每位接牌的玩家拥有**独立的2分钟**倒计时，不累积

## 五、AI 机器人规则

### 5.1 AI 类型

| 难度 | 行为特征 |
|------|---------|
| 🟢 简单 | 随机出牌，不策略思考 |
| 🟡 普通 | 优先出功能牌，会基本策略 |
| 🔴 困难 | 全策略计算，记忆已出牌，最优决策 |

### 5.2 AI 加入规则

- **等待阶段**：游戏开始前，房主可随时添加/移除 AI，可随时调整AI难度
- **游戏中断线**：真人玩家断线超过 **2分钟**，**自动**由 AI 接管，维持游戏继续
- **游戏中主动退出**：玩家点击离开，**立即**由 AI 接管，不判负
- **AI 托管期间**：该座位的手牌对其他玩家**正常显示张数**，但对原断线玩家**隐藏**（防作弊）
- **AI 接管后重连**：原玩家可在游戏结束前重连，**恢复控制权**，AI 退出，恢复手牌可见
- **游戏进行中**：**不可添加新 AI**（仅接管断线/退出玩家），**不可调整AI难度**
- **AI 数量**：真人玩家至少1人，AI 最多补足至8人
- **多个AI连续出牌**：每位AI行动间隔 **1秒**，逐一显示，方便玩家观察

### 5.3 AI 决策时间

| 难度 | 决策延迟 | 说明 |
|------|---------|------|
| 🟢 简单 | 2-4秒 | 模拟"思考"，增加真实感 |
| 🟡 普通 | 1-3秒 | 适度思考 |
| 🔴 困难 | 0.5-1秒 | 快速决策 |

### 5.4 AI 行为逻辑（普通难度）

```
1. 优先出与弃牌颜色相同的牌（含多牌同出）
2. 无同色时，出相同数字/类型
3. 无同色且同数字时，优先出功能牌（跳过/反转/+2）
4. 仍无可出牌时，检查是否满足+4使用条件（手牌无当前颜色），满足则出+4
5. 都不满足时，执行摸牌
6. 剩余1张牌时自动喊 UNO（100%准确率）
   - 时机：出倒数第2张牌后**立即**喊UNO
```

### 5.5 AI 抢打与多牌同出

| 难度 | 抢打策略 | 多牌同出策略 | 限制 |
|------|---------|-------------|------|
| 🟢 简单 | 从不抢打 | 从不使用多牌同出 | 无 |
| 🟡 普通 | 30%概率抢打（有牌时） | 优先多牌同出相同数字的功能牌（如多张+2） | 无 |
| 🔴 困难 | 70%概率抢打（有牌时） | 最优策略：多牌同出+抢打组合，快速获胜 | **每轮最多抢打1次**，防止连续抢打导致玩家无法出牌 |

## 六、胜利与结算

### 6.1 胜利条件

- 首个出完手中所有卡牌的玩家获胜
- **最后一张是功能牌的特殊处理**：
  - 若为 **+2/+4**：下家仍需摸牌并跳过回合，然后游戏结束
    - 若此时**牌堆已空**：触发极端兜底，游戏立即结束，无需摸牌
  - 若为 **跳过/反转**：效果仍生效，但不影响获胜者
  - 若为 **变色**：正常选择颜色，游戏结束
- 游戏立即结束，进入结算界面

### 6.2 计分规则（可选）

| 卡牌类型 | 分值 |
|---------|------|
| 数字牌（0-9）| 面值 |
| 功能牌（跳过/反转/+2）| 20分 |
| 万能牌（变色/+4）| 50分 |

- 获胜者获得**其他所有玩家**手牌分值总和（获胜者自己手牌为空，不计入）
- **计分模式开启时排名**：按剩余手牌分值从低到高排名
- **计分模式关闭时排名**：按出完牌顺序排名（先出完排名高）
- **平局处理**：分值相同则**并列排名**（默认），可配置为按出完牌时间先后排序
- 多局游戏可累计总分排名

### 6.3 结束选项

- **再来一局**：保留房间，重置牌局，玩家不换座，**原房主继续担任房主**
- **解散房间**：所有人返回首页

## 七、特殊机制

### 7.1 断线/退出/重连

| 场景 | 处理方案 |
|------|---------|
| 短暂断线（< 2分钟）| 保留座位，可重连恢复。重连后服务器推送完整游戏状态（手牌、弃牌、玩家状态等）|
| 长时间断线（≥ 2分钟）| 转为 AI 托管，游戏继续 |
| 玩家主动退出 | 立即转为 AI 托管，不判负 |
| AI 托管后重连 | 原玩家恢复控制权，AI 退出 |
| 游戏结束后断线 | 正常结算，不影响结果 |

### 7.2 操作冲突处理

#### 服务器优先原则
所有操作以**服务器接收时间戳**为准，按顺序处理：

| 冲突场景 | 处理方案 |
|---------|---------|
| 玩家A出牌 vs 玩家B质疑 | 出牌先接收则质疑无效（因为手牌已变）；质疑先接收则出牌后检查是否违规 |
| 多人同时质疑 | 只接受第一个到达服务器的质疑，其他返回"已处理" |
| 超时摸牌 vs 玩家主动出牌 | 以先到达的操作为准，后到达的操作被拒绝并提示"回合已结束" |
| 喊UNO vs 质疑 | 以服务器时间戳为准，喊UNO先则质疑失败，质疑先则检查是否已喊 |

### 7.3 账号与设备

| 场景 | 处理方案 |
|------|---------|
| 多设备登录同一房间 | **禁止**：新设备连接时，踢出旧设备连接，提示"已在其他设备登录" |
| 多设备不同房间 | 允许，互不影响 |
| 同一浏览器多标签页 | 视为同一连接，后打开的标签页接管 |

### 7.4 房主权限

- 开始游戏（总人数需至少2人，可以是1真人+1AI）
- 添加/移除 AI
- 踢出玩家
- 解散房间
- **房主退出**：房主断线或退出时，房主身份**自动转让**给最早加入的其他真人玩家
  - 若无其他真人玩家（只剩AI）：房主身份保留为"系统"，直到有新玩家加入
  - 新玩家加入时，自动成为房主

### 7.5 游戏设置（可选）

- 叠加规则开关（+2/+4 是否可叠）
- 强制出牌（摸牌后可出必须出）
- 计分模式开关
- 多牌同出开关（是否允许多牌同出）
- 抢打出牌开关（是否允许抢打）

### 7.6 暂停机制

- **不支持暂停**（防止恶意挂机）
- 玩家需要暂时离开时，可由AI托管或主动退出
- 房主可踢出挂机玩家

### 7.7 房间管理补充

| 场景 | 处理方案 |
|------|---------|
| 房间过期 | 等待阶段 **30分钟** 无操作自动解散；游戏中不自动解散 |
| 玩家重进房间 | **同一会话**（session未过期）：自动回到原座位；**新会话**：视为新玩家，需重新输入房间号 |

### 7.8 UI/UX 规范

| 元素 | 规范 |
|------|------|
| **手牌显示** | 横向滚动或堆叠显示；超过15张时自动缩小卡片尺寸；超过20张显示提示"手牌过多" |
| **手牌状态提示** | **可出牌**：高亮+白色发光边框；**不可出牌**：置灰50%+透明度70%；点击不可出牌显示提示 |
| **分类排序** | 支持按颜色/数字/类型/智能排序；默认按颜色分组显示 |
| **智能推荐** | AI推荐最优解（💡标记），可一键查看所有可出牌 |
| **其他玩家手牌** | 显示剩余张数，不显示具体牌面 |
| **倒计时显示** | 进度条形式，剩余30秒变红色，最后10秒播放音效提醒 |
| **颜色选择弹窗** | 出变色/+4时弹出，**阻塞操作**，必须选择颜色才能继续 |
| **操作反馈** | 出牌：音效+卡牌飞入动画；摸牌：音效+卡牌添加到手中；胜利：特效+音效 |
| **移动端适配** | 触摸优化，长按查看卡牌详情，滑动出牌 |

### 7.9 边界情况处理

| 场景 | 处理方案 |
|------|---------|
| 房间只剩1人 | 游戏暂停，等待新玩家加入；**30秒后无新玩家自动解散房间** |
| 所有真人玩家断线 | AI继续游戏直到结束，或暂停等待（ configurable ） |
| 同时出完最后一张牌 | 以服务器时间戳为准，先到达者获胜，后者为第二名（网络延迟下理论上不可能完全同时）|
| 浏览器刷新 | 同一会话自动重连回原座位，不同会话视为新玩家 |
| 服务器重启 | 游戏状态丢失，所有房间解散，玩家返回首页 |
