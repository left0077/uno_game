## ✅ 已修复的 P0 问题

### P0-1: +2/+4牌未设置pendingDraw
- **文件**: `server/src/game/modes/BaseGameMode.ts`
- **问题**: 打出+2/+4后，`pendingDraw`和`pendingDrawType`没有被设置，叠加规则失效
- **修复**: 在`executePlayCard`中添加逻辑，打出+2/+4时自动设置`pendingDraw`和`pendingDrawType`
- **状态**: ✅ 所有测试通过

### P0-2: +4使用条件验证
- **文件**: `docs/GAME_RULES.md`
- **变更**: 放宽规则，不再强制验证+4使用条件，玩家可以随时使用+4
- **状态**: ✅ 规则书已更新

### P0-3: 超时处理不正确
- **文件**: `server/src/game/UnoGame.ts`, `server/src/shared/index.ts`
- **问题**: 超时后转为AI托管，不是自动摸牌
- **修复**: 
  - 添加`justDrewByTimeout`状态字段
  - 超时后自动摸牌并切换回合（不再转托管）
  - 添加`getNextPlayerId`辅助方法
- **状态**: ✅ 所有测试通过

### P0-4: 摸牌后无法选择出牌
- **状态**: 🔄 移至 v1.1.0 版本
- **原因**: 需要较大的功能改动（状态管理 + 前端UI），单独处理

### P0-5: 多副牌支持
- **文件**: `server/src/game/modes/BaseGameMode.ts`
- **问题**: 5-8人只使用1副牌，规则要求2副
- **修复**: 根据玩家人数创建多副牌（2-4人=1副，5-8人=2副，9-12人=3副）
- **状态**: ✅ 所有测试通过

### P0-6: 摸牌后未检查手牌上限
- **文件**: `server/src/game/modes/OutMode.ts`
- **问题**: Out模式摸牌后未检查手牌上限，玩家可能超过20张而不被淘汰
- **修复**: 
  - 重写`drawCardsForPlayer`方法，在摸牌后调用`checkHandLimit`
  - 修复`checkHandLimit`日志bug（先清空后输出，永远显示0）
- **状态**: ✅ 所有测试通过

### P0-7: 断线2分钟自动托管
- **文件**: `server/src/rooms/RoomManager.ts`
- **问题**: 真人玩家断线超过2分钟没有自动转为托管
- **修复**: 
  - 添加`checkDisconnectedPlayers`方法，每30秒检查一次
  - 断线超过2分钟的玩家自动转为托管（`isAI=true, aiType='host'`）
  - 启动定时器自动执行检查
- **状态**: ✅ 所有测试通过

### P0-8: 主动退出未转托管
- **文件**: `server/src/socket/SocketHandler.ts`
- **问题**: 玩家主动离开时直接移除，不是转为托管
- **修复**: 修改`LEAVE_ROOM`事件处理，游戏进行中时转为托管而不是移除
- **状态**: ✅ 所有测试通过

### P0-9: 重连未清除托管状态
- **文件**: `server/src/socket/SocketHandler.ts`
- **问题**: 重连时设置了`isConnected=true`，但没有清除`isAI`、`aiType`、`aiDifficulty`
- **修复**: 在`player:reconnect`事件中清除AI相关字段，恢复为真人玩家
- **状态**: ✅ 所有测试通过
