## 二、技术方案

```
┌─────────────────────────────────────────────────────────────┐
│                        技术架构                              │
├─────────────────────────────────────────────────────────────┤
│  前端 (Client)          后端 (Server)         通信层         │
│  ┌─────────────┐       ┌─────────────┐      ┌──────────┐   │
│  │ React + TS  │◄─────►│ Node.js     │◄────►│Socket.IO │   │
│  │ TailwindCSS │       │ Express     │      │ WebSocket│   │
│  │ 响应式布局   │       │ 游戏状态管理 │      │ 实时同步  │   │
│  └─────────────┘       └─────────────┘      └──────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 技术选型理由

- **React + TypeScript**：组件化开发，类型安全，生态丰富
- **TailwindCSS**：快速实现响应式UI，移动端适配简单
- **Socket.IO**：实时双向通信，自动降级兼容，房间管理方便
- **内存状态管理**：Uno游戏房间轻量，无需持久化数据库

## 四、Uno游戏规则实现

### 标准规则支持

- 数字牌（0-9，4色：红、黄、绿、蓝）
- 功能牌：跳过、反转、+2
- 万能牌：变色、+4
- 出牌规则：颜色/数字匹配
- 惩罚：忘记喊UNO被质疑时摸2张
- 胜利条件：出完所有牌

### 游戏流程

```
创建房间 → 等待玩家（1-8人）→ 房主开始 → 发牌（每人7张）→ 轮流出牌 → 游戏结束 → 展示排名
```
