## 一、设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **简洁直观** | 核心操作不超过2步，新手无需教程即可上手 |
| **响应优先** | 所有操作100ms内响应，动画流畅60fps |
| **多端一致** | 核心交互逻辑一致，适配不同屏幕尺寸 |
| **反馈明确** | 每次操作都有视觉/音效/震动反馈 |

### 1.2 色彩系统

```css
/* 主色调 */
--color-primary: #6366f1;      /* 靛蓝 - 主按钮、高亮 */
--color-secondary: #8b5cf6;    /* 紫色 - 次要按钮 */
```

## 三、响应式适配方案

### 3.1 断点定义

| 设备 | 宽度 | 布局 |
|------|------|------|
| 手机 | < 640px | 竖屏单列 |
| 平板 | 640-1024px | 自适应网格 |
| PC | > 1024px | 横屏多列 |

### 3.2 手牌区域适配

```
PC端 (>1024px):
- 手牌横向排列
- 最多显示15张不重叠
- 超过15张开始重叠堆叠
- 悬停展开

手机端 (<640px):
- 手牌横向滚动
- 每次显示3-4张
- 左右箭头切换
- 点击选中，点击出牌
```

### 3.3 对手区域适配

```
PC端:
- 顶部横向排列所有对手
- 显示头像、昵称、剩余张数
- 倒计时进度条

手机端:
- 顶部横向滚动
- 只显示头像和张数
- 当前回合高亮
```
