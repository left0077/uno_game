## 🔴 P0 - 严重问题（影响游戏进行）

### 1. +2/+4牌未正确设置叠加状态 [标准UNO]
- **状态**: ❌ 未实现
- **位置**: `server/src/game/modes/BaseGameMode.ts:256-278`
- **问题**: 打出+2/+4后，`pendingDraw`和`pendingDrawType`没有被设置，叠加规则完全失效
- **修复**: 在`executePlayCard`中添加设置逻辑

### 2. ~~+4使用条件验证~~ [标准UNO] ✅ 无需修复
- **状态**: ✅ 按新规则正确实现
- **说明**: 规则已放宽，不再强制验证+4使用条件。玩家可以随时使用+4。

### 3. 超时处理不正确 [标准UNO]
- **状态**: 🐛 有Bug
- **位置**: `server/src/game/UnoGame.ts:153-181`
- **问题**: 超时后转为AI托管，不是自动摸牌；规则要求超时应自动摸牌且摸到后**不可立即打出**
- **修复**: 超时后自动执行摸牌，设置标志禁止立即出牌

### 4. 摸牌后无法选择立即出牌 [标准UNO]
- **状态**: ❌ 未实现
- **位置**: `server/src/game/modes/BaseGameMode.ts:339-356`
- **问题**: 当前实现摸牌后自动结束回合，没有给玩家选择机会
- **修复**: 添加`justDrewCard`状态，保留当前玩家回合（如果摸到可出牌）

### 5. 多副牌支持缺失 [Out模式]
- **状态**: ❌ 未实现
- **位置**: `server/src/game/Card.ts:6-49`
- **问题**: 只创建1副牌（108张），规则要求5-8人=2副，9-12人=3副
- **修复**: 根据人数循环创建多副牌

### 6. 摸牌后未检查手牌上限 [Out模式]
- **状态**: ⚠️ 部分实现
- **位置**: `server/src/game/modes/OutMode.ts:415-432`
- **问题**: `checkHandLimit`只在`executePlayCard`后调用，在摸牌后没有检查
- **修复**: 在`drawCardsForPlayer`后添加检查

### 7. ~~惩罚卡未强制立即使用~~ [Out模式] ✅ 无需修复
- **状态**: ✅ 按新规则正确实现
- **说明**: 经确认，惩罚卡**可以保留**在手牌中，在合适时机主动打出。规则书已更新。

### 8. 被+时连打响应 [Out模式] 🔄 规则已更新
- **状态**: ⚠️ 需要按新规则实现
- **位置**: `server/src/game/modes/OutMode.ts`
- **新规则**: 
  - 被+时可以出连打响应
  - 对子：惩罚不变，传给下家
  - 三条：惩罚不变，跳过1人后的下家受罚
  - 彩虹：惩罚+3，指定任意存活玩家作为新目标
  - 顺子：惩罚+(N-2)，传给下家
- **修复**: 
  1. 在`getAvailableActions`中允许被+时出连打
  2. 修改`handlePendingDraw`支持连打响应
  3. 添加连打响应的效果处理（目标指定、跳过等）
